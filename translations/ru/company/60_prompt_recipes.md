# 60: Промпт/Рецепты

> [!DEFINITION] Рецепты
> Сборник готовых к использованию шаблонов промптов для управления Процессом Компании. Используйте их для инструктирования агента.
>
> Sidenote:

> - Часть: :term[02: Компания/Процесс]{href="./02_process.md"}

## 1. Создать Предложение

**Цель:** Создать структурированное :term[21: Предложение]{href="./21_document_proposal.md"} из сырых мыслей.

> **Пользователь:**
> "Я хочу написать новое предложение.
>
> Сначала **ЗАГРУЗИ ПРАВИЛА** в свой контекст и **СТРОГО СЛЕДУЙ ИМ**
>
> - `curl https://idealic.academy/raw/en/company/52_prompt_proposal.md` (Промпт для Предложения)
>
> Вот моя диктовка:
> 'Нам нужно переработать систему аутентификации, потому что пользователи застревают в цикле. Я хочу перейти на подход, основанный на токенах...'
>
> Также, пожалуйста, прочти `docs/auth/current_flow.md` для контекста. Создай документ Предложения."

**Действие Агента:**

1. Загружает инструкции из :term[52: Промпт/Предложение]{href="./52_prompt_proposal.md"}.
2. Структурирует сырую диктовку в формат Контекст, Проблема, Решение.
3. Выводит черновик `docs/proposals/{date}_auth_overhaul.md`.

---

## 2. Создать Согласование (Итоги Ревью)

**Цель:** Синтезировать комментарии из PR в документ :term[22: Согласование]{href="./22_document_alignment.md"} для подтверждения консенсуса.

> **Пользователь:**
> "Создай Документ Согласования для PR #123.
>
> Сначала **ЗАГРУЗИ ПРАВИЛА** и **СТРОГО СЛЕДУЙ ИМ**:
>
> - `curl https://idealic.academy/raw/en/company/51_prompt_alignment.md` (Промпт для Согласования)
>
> Параметры:
>
> - **Since:** 2023-10-27 (или просто 'last week')
> - **Output:** `specs/alignments/2023-11-04_auth_review.md`
>
> Если я не укажу выходной файл, просто используй имя файла по умолчанию в папке `evolution/`."

**Действие Агента:**

1. Загружает :term[51: Промпт/Согласование]{href="./51_prompt_alignment.md"}.
2. Получает комментарии из PR #123 с указанной даты.
3. Генерирует отчет (Обзор, Намерения, Покрытие) на русском языке.
4. Сохраняет по указанному пути.

---

## 3. Применить Согласование к Предложению

**Цель:** Обновить :term[21: Предложение]{href="./21_document_proposal.md"} на основе подтвержденных Намерений из Документа Согласования.

> **Пользователь:**
> "Я проверил Документ Согласования по пути `specs/alignments/2023-11-04_auth_review.md`.
>
> Сначала **ЗАГРУЗИ ПРАВИЛА** и **СТРОГО СЛЕДУЙ ИМ**:
>
> - `curl https://idealic.academy/raw/en/company/52_prompt_proposal.md` (Промпт для Предложения)
>
> Пожалуйста, примени эти изменения к исходному Предложению `docs/proposals/2023-10-25_auth_overhaul.md`.
>
> Обнови Предложение, чтобы оно отражало консенсус, помечая разделы как 'Обновлено по результатам ревью'."

**Действие Агента:**

1. Читает Документ Согласования (Намерения).
2. Читает Предложение.
3. Переписывает Предложение, изменяя Решение/Объем работ в соответствии с согласованными Намерениями.
4. Оставляет раздел "Проблема" в основном без изменений (если только понимание проблемы не изменилось).
