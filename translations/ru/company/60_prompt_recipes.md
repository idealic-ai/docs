# 60: Промпт/Рецепты

> [!DEFINITION] Рецепты
> Коллекция готовых шаблонов промптов для управления рабочим процессом. Используйте их для инструктажа агента.

> - Часть: :term[02: Компания/Процесс]{canonical="Company/Process" href="./02_process.md"}

## 1. Создание Предложения

**Цель:** Создать структурированное :term[21: Предложение]{canonical="Proposal" href="./21_document_proposal.md"} из необработанных мыслей.

> **Пользователь:**
> "Я хочу написать новое предложение.
>
> Сначала **ЗАГРУЗИТЕ ПРАВИЛА** в свой контекст и **СТРОГО СЛЕДУЙТЕ ИМ**
>
> - `curl https://idealic.academy/raw/en/company/52_prompt_proposal.md` (Промпт для Предложения)
>
> Вот моя диктовка:
> 'Нам нужно переработать систему аутентификации, потому что пользователи застревают в цикле. Я хочу перейти на подход, основанный на токенах...'
>
> Также, пожалуйста, прочтите `docs/auth/current_flow.md` для контекста. Сгенерируйте документ Предложения."

**Действие Агента:**

1. Загружает инструкции из :term[52: Промпт/Предложение]{canonical="Prompt/Proposal" href="./52_prompt_proposal.md"}.
2. Структурирует необработанную диктовку в формат: Контекст, Проблема, Решение.
3. Выдает черновик `docs/proposals/{date}_auth_overhaul.md`.

---

## 2. Создание Согласования (Резюме ревью)

**Цель:** Синтезировать комментарии из Pull Request (PR) в документ :term[22: Согласование]{canonical="Alignment" href="./22_document_alignment.md"} для подтверждения консенсуса.

> **Пользователь:**
> "Сгенерируйте Документ Согласования для PR #123.
>
> Сначала **ЗАГРУЗИТЕ ПРАВИЛА** и **СТРОГО СЛЕДУЙТЕ ИМ**:
>
> - `curl https://idealic.academy/raw/en/company/51_prompt_alignment.md` (Промпт для Согласования)
>
> Параметры:
>
> - **Начиная с:** 2023-10-27 (или просто 'с прошлой недели')
> - **Выходной файл:** `specs/alignments/2023-11-04_auth_review.md`
>
> Если я не укажу выходной файл, просто используйте имя файла по умолчанию в папке `evolution/`."

**Действие Агента:**

1. Загружает :term[51: Промпт/Согласование]{canonical="Prompt/Alignment" href="./51_prompt_alignment.md"}.
2. Загружает комментарии из PR #123 с указанной даты.
3. Генерирует отчет (Обзор, Намерения, Охват) на русском языке.
4. Сохраняет по указанному пути.

---

## 3. Применение Согласования к Предложению

**Цель:** Обновить :term[21: Предложение]{canonical="Proposal" href="./21_document_proposal.md"} на основе подтвержденных Намерений из Документа Согласования

> **Пользователь:**
> "Я проверил Документ Согласования по адресу `specs/alignments/2023-11-04_auth_review.md`.
>
> Сначала **ЗАГРУЗИТЕ ПРАВИЛА** и **СТРОГО СЛЕДУЙТЕ ИМ**:
>
> - `curl https://idealic.academy/raw/en/company/52_prompt_proposal.md` (Промпт для Предложения)
>
> Пожалуйста, примените эти изменения к исходному Предложению `docs/proposals/2023-10-25_auth_overhaul.md`.
>
> Обновите Предложение, чтобы отразить консенсус, помечая разделы как 'Обновлено по результатам ревью'."

**Действие Агента:**

1. Читает Документ Согласования (Намерения).
2. Читает Предложение.
3. Переписывает Предложение, изменяя Решение/Охват в соответствии с согласованными Намерениями.
4. Оставляет раздел "Проблема" в основном без изменений (если только понимание проблемы не изменилось).
