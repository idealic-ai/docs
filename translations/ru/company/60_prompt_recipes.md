# 60: Промпт/Рецепты

> [!DEFINITION] Рецепты
> Коллекция готовых шаблонов :term[промптов]{canonical="prompt"} для управления :term[Процессом Компании]{canonical="Company Process"}. Используйте их, чтобы давать инструкции :term[агенту]{canonical="agent"}.
>
> Sidenote:
> - Часть: :term[02: Компания/Процесс]{canonical="Company/Process" href="./02_process.md"}

## 1. Создать Предложение

**Цель:** Сгенерировать структурированное :term[21: Предложение]{canonical="Proposal" href="./21_document_proposal.md"} из сырых мыслей.

> **Пользователь:**
> «Я хочу написать новое предложение.
>
> Сначала **ЗАГРУЗИ ПРАВИЛА** в свой контекст:
>
> - `curl https://idealic.academy/raw/simple-ru/company/52_prompt_proposal.md` (Промпт для Предложения)
>
> Вот моя диктовка:
> 'Нам нужно переработать систему аутентификации, потому что пользователи застревают в цикле. Я хочу перейти на подход с использованием токенов...'
>
> Также для контекста прочти `docs/auth/current_flow.md`. Сгенерируй документ :term[Предложения]{canonical="Proposal"}.»

**Действие Агента:**

1. Загружает инструкции из :term[52: Промпт/Предложение]{canonical="Prompt/Proposal" href="./52_prompt_proposal.md"}.
2. Структурирует сырую диктовку в формат: Контекст, Проблема, Решение.
3. Выводит черновик `docs/proposals/{date}_auth_overhaul.md`.

## 2. Сгенерировать Согласование (Итоги Ревью)

**Цель:** Обобщить комментарии из PR в документ :term[22: Согласование]{canonical="Alignment" href="./22_document_alignment.md"} для подтверждения консенсуса.

> **Пользователь:**
> «Сгенерируй Документ Согласования для PR #123.
>
> Сначала **ЗАГРУЗИ ПРАВИЛА**:
>
> - `curl https://idealic.academy/raw/simple-ru/company/51_prompt_alignment.md` (Промпт для Согласования)
>
> Параметры:
>
> - **Начиная с:** 2023-10-27 (или просто «прошлая неделя»)
> - **Вывод:** `specs/alignments/2023-11-04_auth_review.md`
>
> Если я не укажу путь для вывода, просто используй имя файла по умолчанию в папке `evolution/`.»

**Действие Агента:**

1. Загружает :term[51: Промпт/Согласование]{canonical="Prompt/Alignment" href="./51_prompt_alignment.md"}.
2. Получает комментарии из PR #123 с указанной даты.
3. Генерирует отчёт (Обзор, :term[Намерения]{canonical="Intents"}, Покрытие) на русском языке.
4. Сохраняет по указанному пути.

## 3. Применить Согласование к Предложению

**Цель:** Обновить :term[21: Предложение]{canonical="Proposal" href="./21_document_proposal.md"} на основе утвержденных :term[Намерений]{canonical="Intents"} из Документа Согласования.

> **Пользователь:**
> «Я проверил Документ Согласования по адресу `specs/alignments/2023-11-04_auth_review.md`.
>
> Сначала **ЗАГРУЗИ ПРАВИЛА**:
>
> - `curl https://idealic.academy/raw/simple-ru/company/52_prompt_proposal.md` (Промпт для Предложения)
>
> Пожалуйста, примени эти изменения к исходному :term[Предложению]{canonical="Proposal"} `docs/proposals/2023-10-25_auth_overhaul.md`.
>
> Обнови :term[Предложение]{canonical="Proposal"}, чтобы оно отражало консенсус, помечая разделы как „Обновлено по результатам ревью“.»

**Действие Агента:**

1. Читает Документ Согласования (:term[Намерения]{canonical="Intents"}).
2. Читает :term[Предложение]{canonical="Proposal"}.
3. Переписывает :term[Предложение]{canonical="Proposal"}, изменяя Решение/Объём работ в соответствии с согласованными :term[Намерениями]{canonical="Intents"}.
4. Оставляет раздел «Проблема» в основном без изменений (если только понимание проблемы не изменилось).
