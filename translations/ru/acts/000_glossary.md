# Глоссарий Терминов

Этот документ содержит определения ключевых понятий, используемых в Протоколе Идей и связанных с ним системах.

- :dfn[Идея]{canonical="Idea" href="./101_concept_idea.md"}: Самодостаточный, сохраняющий состояние триплет данных (`schema`, `solution`, `context`), представляющий собой единицу знания. Это постоянный вычислительный примитив, а не эфемерный промпт.

  > Sidenote:
  > - [101: Концепция/Идея](./101_concept_idea.md)

- :dfn[Идеатор]{canonical="Ideator" href="./103_concept_ideator.md"}: :term[Идея]{canonical="Idea"}, которая принимает входные данные, что обозначается контекстным сообщением с `type: "input"`. Он действует как функция, преобразуя входные данные в выходные.

  > Sidenote:
  > - [103: Концепция/Идеатор](./103_concept_ideator.md)

- :dfn[Трансформер Идей]{canonical="Idea Transformer" href="./103_concept_ideator.md"}: Специализированный :term[Идеатор]{canonical="Ideator"}, который принимает другую :term[Идею]{canonical="Idea"} в качестве входных данных.

  > Sidenote:
  > - [103: Концепция/Идеатор](./103_concept_ideator.md)

- :dfn[Входное Сообщение]{canonical="Input Message" href="./007_agent_input.md"}: Контекстное сообщение, содержащее `schema` и данные `input`. Оно определяет ожидаемые входные данные для :term[Запроса]{canonical="Request"}, делая его повторно используемым, подобным функции компонентом.

  > Sidenote:
  > - [007: Агент/Ввод](./007_agent_input.md)

- :dfn[Инструмент]{canonical="Tool" href="./002_agent_tool.md"}: Схема, определяющая возможность, которую может использовать агент. Она представляется LLM как часть запроса, действуя как структурированный интерфейс для потенциального действия. LLM активирует инструмент, генерируя :term[Вызов]{canonical="Call"} с конкретными параметрами, который затем выполняется либо скрыто LLM, либо явно зарегистрированной функцией кода (:term[Действие]{canonical="Activity"}).

  > Sidenote:
  > - [002: Агент/Инструмент](./002_agent_tool.md)

- :dfn[Действие]{canonical="Activity" href="./003_agent_activity.md"}: Явная, детерминированная функция кода, которая реализует логику для :term[Инструмента]{canonical="Tool"}. Это механизм для выполнения действий, требующих внешних вызовов API, операций с базами данных или любой задачи, которая не может быть обработана в скрытом пространстве LLM.

  > Sidenote:
  > - [003: Агент/Действие](./003_agent_activity.md)

- :dfn[Вызов]{canonical="Call" href="./004_agent_call.md"}: Конкретный, исполняемый экземпляр :term[Инструмента]{canonical="Tool"} с конкретными значениями для его `params`. Это ориентированный на вызов запрос о том, что \_должно быть сделано\_.

  > Sidenote:
  > - [004: Агент/Вызов](./004_agent_call.md)

- :dfn[Область видимости]{canonical="Scope" href="./004_agent_call.md"}: Место выполнения :term[Вызова]{canonical="Call"}.
  - :dfn[Встроенная Область видимости]{canonical="Inline Scope" href="./004_agent_call.md"}: :term[Вызов]{canonical="Call"} обрабатывается в контексте текущего агента.
  - :dfn[Делегированная Область видимости]{canonical="Delegate Scope" href="./004_agent_call.md"}: :term[Вызов]{canonical="Call"} делегируется внешнему, изолированному делегату.
    > Sidenote:
    > - [004: Агент/Вызов](./004_agent_call.md)

- :dfn[Метод]{canonical="Method" href="./004_agent_call.md"}: Механизм выполнения для :term[Вызова]{canonical="Call"}.
  - :dfn[Явное Выполнение]{canonical="Explicit Execution" href="./003_agent_activity.md"}: Результат :term[Вызова]{canonical="Call"} генерируется детерминированным кодом (:term[Действие]{canonical="Activity"}).

    > Sidenote:
    > - [003: Агент/Действие](./003_agent_activity.md)

  - :dfn[Скрытое Выполнение]{canonical="Latent Execution" href="./104_concept_latent.md"}: Результат :term[Вызова]{canonical="Call"} генерируется LLM.

    > Sidenote:
    > - [104: Концепция/Скрытое](./104_concept_latent.md)

- :dfn[Делегат]{canonical="Delegate" href="./012_agent_delegate.md"}: Протокол для изоляции контекста выполнения. Вызываемый свойством `_delegate` :term[Вызова]{canonical="Call"}, он выполняет :term[Действие]{canonical="Activity"} или новый :term[Запрос]{canonical="Request"} в среде "чистой комнаты", при этом свойство `_scopes` обеспечивает контролируемый доступ к родительскому контексту.

  > Sidenote:
  > - [012: Агент/Делегат](./012_agent_delegate.md)

- :dfn[Область видимости]{canonical="Scope" href="./013_agent_scopes.md"}: Ключ, который идентифицирует часть контекста из родительской среды, которая будет доступна (`scoped`) для выполнения. Его можно использовать для фокусировки внимания LLM при **Встроенном Скрытом** выполнении или для построения всего контекста для выполнения в **Делегированной Области видимости**. Управляется свойством `_scopes`.

  > Sidenote:
  > - [013: Агент/Области Видимости](./013_agent_scopes.md)

- :dfn[Цикл]{canonical="Loop" href="./005_agent_loop.md"}: Последовательность :term[Запросов]{canonical="Request"}, направленная на достижение цели. Агент продолжает вызывать :term[Запросы]{canonical="Request"}, обрабатывать результирующие :term[Вызовы]{canonical="Call"} и передавать результат обратно в контекст следующего :term[Запроса]{canonical="Request"} до тех пор, пока не перестанут генерироваться :term[Вызовы]{canonical="Call"}.

  > Sidenote:
  > - [005: Агент/Цикл](./005_agent_loop.md)

- :dfn[HITL (Человек-в-цикле)]{canonical="HITL (Human-in-the-Loop)" href="./005_agent_loop.md#human-in-the-loop-hitl"}: Практика вставки контрольной точки с участием человека в автоматизированный процесс. В контексте :term[Цикла Выполнения]{canonical="Execution Loop"} это относится к возможности пользователя одобрить, отклонить или изменить предложенные агентом :term[Вызовы]{canonical="Call"} до их выполнения.

  > Sidenote:
  > - [005: Агент/Цикл](./005_agent_loop.md#human-in-the-loop-hitl)

- :dfn[Эволюция]{canonical="Evolution" href="./106_concept_evolution.md"}: Процесс, посредством которого ИИ-центричная система автономно или полуавтономно адаптируется, улучшается и изменяет свою собственную структуру, логику и возможности с течением времени в ответ на новые данные, обратную связь или изменяющиеся цели.

  > Sidenote:
  > - [106: Концепция/Эволюция](./106_concept_evolution.md)

- :dfn[Сообщение о Состоянии]{canonical="State Message" href="./009_agent_state.md"}: Контекстное сообщение, содержащее объект `state` и необязательную `schema`. Оно представляет собой постоянное состояние, поддерживаемое между тактами цикла выполнения агента.

  > Sidenote:
  > - [009: Агент/Состояние](./009_agent_state.md)

- :dfn[Сообщение с Данными]{canonical="Data Message" href="./006_agent_data.md"}: Постоянное сообщение контекста, содержащее значение `data` и необязательную `schema`. Оно сохраняется на протяжении всего цикла выполнения агента для обеспечения стабильного, структурированного контекста.

- :dfn[План]{canonical="Plan" href="./010_agent_plan.md"}: Контекстное сообщение, несущее граф потока данных :term[Вызовов Инструментов]{canonical="Tool Call"}, который представляет стратегию агента. Оно передается между шагами для обеспечения итеративного выполнения и адаптации.

  > Sidenote:
  > - [010: Агент/План](./010_agent_plan.md)

- :dfn[Идея-Процесс]{canonical="Process Idea" href="./010*agent_plan.md"}: Самодостаточная [Идея](./101_concept_idea.md), которая содержит стратегический [План](./010_agent_plan.md) и его текущее состояние выполнения. Ее `schema` — это библиотека [Инструментов](./002_agent_tool.md), ее `solution` — это \_новый* `План` для текущего такта, а ее `context` содержит `Input`, `State` и предыдущий `План`.

  > Sidenote:
  > - [010: Агент/План](./010_agent_plan.md)

- :dfn[Идея-Носитель]{canonical="Vessel Idea" href="./202_idea_vessel.md"}: Самодостаточная :term[Идея]{canonical="Idea"}, которая является одновременно и определением реактивной способности, и записью выбранной реакции. Ее `schema` определяет всю вселенную возможных :term[Инструментов]{canonical="Tool"}, а ее `solution` фиксирует конкретные :term[Вызовы]{canonical="Call"} (экземпляры этих :term[Инструментов]{canonical="Tool"}), которые были выбраны в ответ на стимул.

  > Sidenote:
  > - [202: Идея/Носитель](./202_idea_vessel.md)

- :dfn[Экземплирование]{canonical="Instancing" href="./011_agent_instancing.md"}: Процесс обработки нескольких независимых :term[Экземпляров]{canonical="Instance"} (каждый со своим уникальным идентификатором и соответствующим сообщением о :term[Состоянии]{canonical="State"}) в рамках одного запроса агента.

  > Sidenote:
  > - [011: Агент/Экземплирование](./011_agent_instancing.md)

- :dfn[Запрос]{canonical="Request" href="./001_agent_request.md"}: Единичный, самодостаточный вызов LLM, который принимает `context` и `schema` и создает `solution`.

- :dfn[Экземпляр]{canonical="Instance" href="./011_agent_instancing.md"}: Единый, уникально идентифицированный объект состояния и связанный с ним контекст, обрабатываемый как один из многих в рамках операции :term[Экземплирования]{canonical="Instancing"}.

  > Sidenote:
  > - [011: Агент/Экземплирование](./011_agent_instancing.md)

- :dfn[Реактор]{canonical="Reactor" href="./303_ideator_reactor.md"}: Специальный :term[Трансформер Идей]{canonical="Idea Transformer"}, разработанный как универсальная среда выполнения для пошаговых взаимодействий агентов с сохранением состояния. Он принимает состояние игры/процесса (:term[Идея]{canonical="Idea"}) и создает следующее состояние (:term[Идея]{canonical="Idea"}).

  > Sidenote:
  > - [303: Идеатор/Реактор](./303_ideator_reactor.md)

- :dfn[Ссылка на Переменную]{canonical="Variable Reference" href="./004_agent_call.md"}: Строка со специальным синтаксисом (`†<kind>.<path>`), используемая в параметрах :term[Вызова Инструмента]{canonical="Tool Call"} для ссылки на значение из контекста (например, из сообщения :term[Состояния]{canonical="State"} или :term[Ввода]{canonical="Input"}). Это позволяет агенту динамически связывать инструменты вместе, используя вывод одного инструмента в качестве входа для другого.

- :dfn[Решение]{canonical="Solution" href="./001_agent_request.md"}: Структурированный объект, возвращаемый :term[Запросом]{canonical="Request"}. Он содержит предложенные :term[Вызовы Инструментов]{canonical="Tool Call"} и конечный `output`, соответствующий предопределенной JSON Schema.
