# Глоссарий терминов

Этот документ содержит определения ключевых понятий, используемых в Протоколе Идей и связанных с ним системах.

- :dfn[Идея]{canonical="Idea" href="./101_concept_idea.md"}: Самодостаточная тройка данных с состоянием (`schema`, `solution`, `context`), которая представляет собой единицу знания. Это постоянный вычислительный элемент, а не временная команда (промпт).

  > Sidenote:
  > - [101: Концепция/Идея](./101_concept_idea.md)

- :dfn[Идеатор]{canonical="Ideator" href="./103_concept_ideator.md"}: :term[Идея]{canonical="Idea"}, которая принимает входные данные, что обозначается контекстным сообщением `type: "input"`. Она действует как функция, преобразуя входные данные в выходные.

  > Sidenote:
  > - [103: Концепция/Идеатор](./103_concept_ideator.md)

- :dfn[Трансформатор Идей]{canonical="Idea Transformer" href="./103_concept_ideator.md"}: Специализированный :term[Идеатор]{canonical="Ideator"}, который принимает в качестве входных данных другую :term[Идею]{canonical="Idea"}.

  > Sidenote:
  > - [103: Концепция/Идеатор](./103_concept_ideator.md)

- :dfn[Входное Сообщение]{canonical="Input Message" href="./007_agent_input.md"}: Контекстное сообщение, содержащее `schema` и входные данные `input`. Оно определяет ожидаемые входные данные для :term[Запроса]{canonical="Request"}, делая его многоразовым, подобным функции, компонентом.

  > Sidenote:
  > - [007: Агент/Ввод](./007_agent_input.md)

- :dfn[Инструмент]{canonical="Tool" href="./002_agent_tool.md"}: Схема, определяющая возможность, которую может использовать агент. Она передаётся LLM как часть запроса и служит структурированным интерфейсом для потенциального действия. LLM активирует инструмент, создавая :term[Вызов]{canonical="Call"} с определёнными параметрами, который затем выполняется либо в скрытом виде LLM, либо явно зарегистрированной функцией (:term[Действием]{canonical="Activity"}).

  > Sidenote:
  > - [002: Агент/Инструмент](./002_agent_tool.md)

- :dfn[Действие]{canonical="Activity" href="./003_agent_activity.md"}: Явная, детерминированная функция, реализующая логику для :term[Инструмента]{canonical="Tool"}. Это механизм для выполнения действий, требующих вызовов внешних API, операций с базами данных или любых других задач, которые не могут быть выполнены в скрытом пространстве LLM.

  > Sidenote:
  > - [003: Агент/Действие](./003_agent_activity.md)

- :dfn[Вызов]{canonical="Call" href="./004_agent_call.md"}: Конкретный, исполняемый экземпляр :term[Инструмента]{canonical="Tool"} с определёнными значениями для его `params`. Это запрос, сфокусированный на вызове, о том, что \_должно быть сделано\_.

  > Sidenote:
  > - [004: Агент/Вызов](./004_agent_call.md)

- :dfn[Область видимости]{canonical="Scope" href="./004_agent_call.md"}: Место выполнения :term[Вызова]{canonical="Call"}.
  - **Встроенная область видимости**: :term[Вызов]{canonical="Call"} обрабатывается в контексте текущего агента.
  - **Делегированная область видимости**: :term[Вызов]{canonical="Call"} делегируется внешнему, изолированному делегату.
    > Sidenote:
    > - [004: Агент/Вызов](./004_agent_call.md)

- **Метод**: Механизм выполнения :term[Вызова]{canonical="Call"}.
  - **Явное выполнение**: Результат :term[Вызова]{canonical="Call"} генерируется детерминированным кодом (:term[Действием]{canonical="Activity"}).

    > Sidenote:
    > - [003: Агент/Действие](./003_agent_activity.md)

  - **Скрытое выполнение**: Результат :term[Вызова]{canonical="Call"} генерируется LLM.

    > Sidenote:
    > - [104: Концепция/Скрытое](./104_concept_latent.md)

- :dfn[Делегат]{canonical="Delegate" href="./012_agent_delegate.md"}: Протокол для изоляции контекста выполнения. Вызывается свойством `_delegate` :term[Вызова]{canonical="Call"}, выполняет :term[Действие]{canonical="Activity"} или новый :term[Запрос]{canonical="Request"} в изолированной среде («чистой комнате»), при этом свойство `_scopes` предоставляет контролируемый доступ к родительскому контексту.

  > Sidenote:
  > - [012: Агент/Делегат](./012_agent_delegate.md)

- :dfn[Область видимости]{canonical="Scope" href="./013_agent_scopes.md"}: Ключ, который идентифицирует часть контекста из родительской среды, чтобы сделать её доступной (`scoped`) для выполнения. Он может использоваться для фокусировки внимания LLM при **Встроенном Скрытом** выполнении или для построения всего контекста при выполнении с **Делегированной областью видимости**. Контролируется свойством `_scopes`.

  > Sidenote:
  > - [013: Агент/Области видимости](./013_agent_scopes.md)

- :dfn[Цикл]{canonical="Loop" href="./005_agent_loop.md"}: Последовательность :term[Запросов]{canonical="Request"}, направленная на достижение цели. Агент продолжает вызывать :term[Запросы]{canonical="Request"}, обрабатывать результирующие :term[Вызовы]{canonical="Call"} и передавать результат обратно в контекст следующего :term[Запроса]{canonical="Request"} до тех пор, пока не перестанут генерироваться :term[Вызовы]{canonical="Call"}.

  > Sidenote:
  > - [005: Агент/Цикл](./005_agent_loop.md)

- :dfn[HITL (Человек-в-цикле)]{canonical="HITL (Human-in-the-Loop)" href="./005_agent_loop.md#human-in-the-loop-hitl"}: Практика включения человека для контроля в автоматизированный процесс. В контексте :term[Цикла выполнения]{canonical="Execution Loop"}, это означает возможность для пользователя одобрить, отклонить или изменить предложенные агентом :term[Вызовы]{canonical="Call"} перед их выполнением.

  > Sidenote:
  > - [005: Агент/Цикл](./005_agent_loop.md#human-in-the-loop-hitl)

- :dfn[Эволюция]{canonical="Evolution" href="./106_concept_evolution.md"}: Процесс, посредством которого :term[ИИ-центричная]{canonical="AI-Native"} система автономно или полуавтономно адаптируется, улучшается и изменяет свою собственную структуру, логику и возможности с течением времени в ответ на новые данные, обратную связь или меняющиеся цели.

  > Sidenote:
  > - [106: Концепция/Эволюция](./106_concept_evolution.md)

- :dfn[Сообщение о состоянии]{canonical="State Message" href="./009_agent_state.md"}: Контекстное сообщение, содержащее объект `state` и необязательную `schema`. Оно представляет собой постоянное состояние, сохраняющееся между тактами цикла выполнения агента.

  > Sidenote:
  > - [009: Агент/Состояние](./009_agent_state.md)

- :dfn[Сообщение с данными]{canonical="Data Message" href="./006_agent_data.md"}: Постоянное контекстное сообщение, содержащее значение `data` и необязательную `schema`. Оно сохраняется на протяжении всего цикла выполнения агента для обеспечения стабильного, структурированного контекста.

- :dfn[План]{canonical="Plan" href="./010_agent_plan.md"}: Контекстное сообщение, содержащее граф потока данных из :term[Вызовов Инструментов]{canonical="Tool Call"}, который представляет стратегию агента. Оно передаётся между шагами для обеспечения итеративного выполнения и адаптации.

  > Sidenote:
  > - [010: Агент/План](./010_agent_plan.md)

- :dfn[Процесс-Идея]{canonical="Process Idea" href="./010_agent_plan.md"}: Самодостаточная [Идея](./101_concept_idea.md), которая содержит стратегический [План](./010_agent_plan.md) и его текущее состояние выполнения. Её `schema` — это библиотека [Инструментов](./002_agent_tool.md), её `solution` — это \_новый\_ `План` для текущего такта, а её `context` содержит `Ввод`, `Состояние` и предыдущий `План`.

  > Sidenote:
  > - [010: Агент/План](./010_agent_plan.md)

- :dfn[Идея-Носитель]{canonical="Vessel Idea" href="./202_idea_vessel.md"}: Самодостаточная `Идея`, которая является одновременно определением реактивной способности и записью её выбранной реакции. Её `schema` определяет всю вселенную возможных :term[Инструментов]{canonical="Tool"}, а её `solution` фиксирует конкретные :term[Вызовы]{canonical="Call"} (экземпляры этих :term[Инструментов]{canonical="Tool"}), которые были выбраны в ответ на стимул.

  > Sidenote:
  > - [202: Идея/Носитель](./202_idea_vessel.md)

- :dfn[Экземплирование]{canonical="Instancing" href="./011_agent_instancing.md"}: Процесс обработки нескольких независимых :term[Экземпляров]{canonical="Instance"} (каждый со своим уникальным идентификатором и соответствующим :term[сообщением о Состоянии]{canonical="State Message"}) в рамках одного запроса агента.

  > Sidenote:
  > - [011: Агент/Экземплирование](./011_agent_instancing.md)

- :dfn[Запрос]{canonical="Request" href="./001_agent_request.md"}: Единый, самодостаточный вызов LLM, который принимает `context` и `schema` и создаёт `solution`.

- :dfn[Экземпляр]{canonical="Instance" href="./011_agent_instancing.md"}: Единый, уникально идентифицированный объект состояния и связанный с ним контекст, обрабатываемый как один из многих в операции :term[Экземплирования]{canonical="Instancing"}.

  > Sidenote:
  > - [011: Агент/Экземплирование](./011_agent_instancing.md)

- :dfn[Реактор]{canonical="Reactor" href="./303_ideator_reactor.md"}: Особый :term[Трансформатор Идей]{canonical="Idea Transformer"}, разработанный как универсальная среда выполнения для пошаговых взаимодействий агентов с сохранением состояния. Он принимает состояние игры/процесса (:term[Идею]{canonical="Idea"}) и создаёт следующее состояние (:term[Идею]{canonical="Idea"}).

  > Sidenote:
  > - [303: Идеатор/Реактор](./303_ideator_reactor.md)

- **Ссылка на переменную**: Строка со специальным синтаксисом (`†<тип>.<путь>`), используемая в параметрах :term[Вызова Инструмента]{canonical="Tool Call"} для ссылки на значение из контекста (например, из :term[сообщения о Состоянии]{canonical="State Message"} или :term[Входного сообщения]{canonical="Input Message"}). Это позволяет агенту динамически связывать инструменты, используя выходные данные одного инструмента в качестве входных для другого.

- :dfn[Решение]{canonical="Solution" href="./001_agent_request.md"}: Структурированный объект, возвращаемый :term[Запросом]{canonical="Request"}. Он содержит предложенные :term[Вызовы Инструментов]{canonical="Tool Call"} и конечный `output`, соответствующий предопределённой JSON Schema.
