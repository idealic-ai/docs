# Глоссарий терминов

Этот документ содержит определения ключевых понятий, используемых в протоколе Idea и связанных с ним системах.

- :dfn[Идея]{canonical="Idea" href="./101_concept_idea.md"}: Самодостаточный, сохраняющий состояние триплет данных (`schema`, `solution`, `context`), представляющий собой единицу знания. Это постоянный вычислительный примитив, а не эфемерный промпт.

  > Sidenote:
  > - [101: Концепция/Идея](./101_concept_idea.md)

- :dfn[Идеатор]{canonical="Ideator" href="./103_concept_ideator.md"}: Это :term[Идея]{canonical="Idea"}, которая принимает входные данные, что обозначается контекстным сообщением с `type: "input"`. Он действует как функция, преобразующая входные данные в выходные.

  > Sidenote:
  > - [103: Концепция/Идеатор](./103_concept_ideator.md)

- :dfn[Трансформер Идей]{canonical="Idea Transformer" href="./103_concept_ideator.md"}: Специализированный :term[Идеатор]{canonical="Ideator"}, который принимает другую :term[Идею]{canonical="Idea"} в качестве входных данных.

  > Sidenote:
  > - [103: Концепция/Идеатор](./103_concept_ideator.md)

- :dfn[Входное сообщение]{canonical="Input Message" href="./007_agent_input.md"}: Контекстное сообщение, содержащее `schema` и входные `данные`. Оно определяет ожидаемые входные данные для :term[Запроса]{canonical="Request"}, делая его повторно используемым, подобным функции компонентом.

  > Sidenote:
  > - [007: Агент/Ввод](./007_agent_input.md)

- :dfn[Инструмент]{canonical="Tool" href="./002_agent_tool.md"}: Схема, определяющая возможность, которую может использовать агент. Она представляется LLM как часть запроса, выступая в роли структурированного интерфейса для потенциального действия. LLM активирует инструмент, генерируя :term[Вызов]{canonical="Call"} с определёнными параметрами, который затем исполняется либо в скрытом состоянии самой LLM, либо явно зарегистрированной функцией кода (:term[Действием]{canonical="Activity"}).

  > Sidenote:
  > - [002: Агент/Инструмент](./002_agent_tool.md)

- :dfn[Действие]{canonical="Activity" href="./003_agent_activity.md"}: Явная, детерминированная функция кода, реализующая логику для :term[Инструмента]{canonical="Tool"}. Это механизм для выполнения действий, требующих внешних вызовов API, операций с базами данных или любых задач, которые не могут быть обработаны в скрытом пространстве LLM.

  > Sidenote:
  > - [003: Агент/Действие](./003_agent_activity.md)

- :dfn[Ветвь]{canonical="Branch" href="./108_concept_visibility.md#branches-publication-and-partitioning"}: Именованная метка, которая разделяет пространство видимости, создавая параллельную, изолированную среду для разработки и экспериментов. Связывание :term[Идеи]{canonical="Idea"} с ветвью является актом публикации.

- :dfn[Вызов]{canonical="Call" href="./004_agent_call.md"}: Конкретный, исполняемый экземпляр :term[Инструмента]{canonical="Tool"} с определёнными значениями для его `params`. Это ориентированный на исполнение запрос того, что \_должно быть сделано\_.

  > Sidenote:
  > - [004: Агент/Вызов](./004_agent_call.md)

- :dfn[Время отсечения]{canonical="Cutoff Time" href="./108_concept_visibility.md#the-cutoff-time-temporal-retrieval"}: Временная метка, сопровождающая запрос на разрешение, которая указывает резолверу найти версию :term[Идеи]{canonical="Idea"}, считавшуюся последней на тот конкретный момент времени.

- :dfn[Метод]{canonical="Method" href="./004_agent_call.md"}: Механизм исполнения :term[Вызова]{canonical="Call"}.
  - :dfn[Явное исполнение]{canonical="Explicit Execution" href="./003_agent_activity.md"}: Результат :term[Вызова]{canonical="Call"} генерируется детерминированным кодом (:term[Действием]{canonical="Activity"}).

    > Sidenote:
    > - [003: Агент/Действие](./003_agent_activity.md)

  - :dfn[Скрытое исполнение]{canonical="Latent Execution" href="./104_concept_latent.md"}: Результат :term[Вызова]{canonical="Call"} генерируется LLM.

    > Sidenote:
    > - [104: Концепция/Скрытое](./104_concept_latent.md)

- :dfn[Делегат]{canonical="Delegate" href="./012_agent_delegate.md"}: Протокол для изоляции контекста исполнения. Вызываемый свойством `_delegate` :term[Вызова]{canonical="Call"}, он исполняет :term[Действие]{canonical="Activity"} или новый :term[Запрос]{canonical="Request"} в среде «чистой комнаты», при этом свойство `_scopes` предоставляет контролируемый доступ к родительскому контексту.

  > Sidenote:
  > - [012: Агент/Делегат](./012_agent_delegate.md)

- :dfn[Область видимости]{canonical="Scope" href="./013_agent_scopes.md"}: Ключ, который идентифицирует часть контекста из родительской среды для предоставления (`scoped`) исполнению. Он может использоваться для фокусировки внимания LLM при **встроенном скрытом** исполнении или для построения всего контекста для исполнения **делегированной области видимости**. Контролируется свойством `_scopes`.

  > Sidenote:
  > - [013: Агент/Области видимости](./013_agent_scopes.md)

- :dfn[Цикл]{canonical="Loop" href="./005_agent_loop.md"}: Последовательность :term[Запросов]{canonical="Request"}, направленная на достижение цели. Агент продолжает вызывать :term[Запросы]{canonical="Request"}, обрабатывать результирующие :term[Вызовы]{canonical="Call"} и передавать результат обратно в контекст следующего :term[Запроса]{canonical="Request"} до тех пор, пока не перестанут генерироваться :term[Вызовы]{canonical="Call"}.

  > Sidenote:
  > - [005: Агент/Цикл](./005_agent_loop.md)

- :dfn[HITL (Человек-в-цикле)]{canonical="HITL (Human-in-the-Loop)" href="./005_agent_loop.md#human-in-the-loop-hitl"}: Практика включения контрольной точки с участием человека в автоматизированный процесс. В контексте :term[Цикла исполнения]{canonical="Execution Loop"} это означает возможность для пользователя утвердить, отклонить или изменить предложенные агентом :term[Вызовы]{canonical="Call"} перед их исполнением.

  > Sidenote:
  > - [005: Агент/Цикл](./005_agent_loop.md#human-in-the-loop-hitl)

- :dfn[Иерархическое версионирование]{canonical="Hierarchical Versioning" href="./108_concept_visibility.md#versioning-creating-the-states-to-be-seen"}: Схема версионирования, в которой версии представляют собой идентификаторы, разделённые точками (например, `1.2.feature-x.3`), объединяющая концепции линейных релизов, ветвей и черновиков в единую иерархическую структуру.

- :dfn[Эволюция]{canonical="Evolution" href="./106_concept_evolution.md"}: Процесс, в ходе которого :term[ИИ-центричная]{canonical="AI-Native"} система автономно или полуавтономно адаптируется, улучшается и изменяет свою собственную структуру, логику и возможности с течением времени в ответ на новые данные, обратную связь или изменяющиеся цели.

  > Sidenote:
  > - [106: Концепция/Эволюция](./106_concept_evolution.md)

- :dfn[Эмерджентная идентичность]{canonical="Emergent Identity" href="./107_concept_identity.md"}: Распознаваемый паттерн, возникающий из сети взаимосвязей, соединяющей коллекцию :term[Идей]{canonical="Idea"}. Идентичность — это не внутреннее свойство отдельной :term[Идеи]{canonical="Idea"}, а качество, которое возникает из её контекста.

- :dfn[idea:]{canonical="idea:" href="./109_concept_addressing.md"}: Схема URI для навигации по версионированной, разветвлённой реальности экосистемы :term[Идей]{canonical="Idea"}. Она предоставляет синтаксис как для простых, динамических запросов, так и для полностью разрешённых, постоянных ссылок.

- :dfn[Отношения идентичности]{canonical="Identity Relationships" href="./107_concept_identity.md#the-relational-fabric-of-identity"}: Набор связей, которые порождают :term[Эмерджентную идентичность]{canonical="Emergent Identity"}.
  - :dfn[Происхождение]{canonical="Lineage"}: Историческая цепь версий, которая связывает :term[Идею]{canonical="Idea"} с её прошлыми и будущими итерациями.
  - :dfn[Причинность]{canonical="Causality"}: Отношение, при котором одна :term[Идея]{canonical="Idea"} напрямую вызывает или инициирует создание другой.
  - :dfn[Группировка]{canonical="Grouping"}: Механизм для объединения нескольких связанных :term[Идей]{canonical="Idea"} в единый, управляемый блок.

- :dfn[Сообщение о состоянии]{canonical="State Message" href="./009_agent_state.md"}: Контекстное сообщение, содержащее объект `state` и необязательную `schema`. Оно представляет собой постоянное состояние, поддерживаемое между тактами цикла исполнения агента.

  > Sidenote:
  > - [009: Агент/Состояние](./009_agent_state.md)

- :dfn[Сообщение с данными]{canonical="Data Message" href="./006_agent_data.md"}: Постоянное контекстное сообщение, содержащее значение `data` и необязательную `schema`. Оно сохраняется на протяжении всего цикла исполнения агента для обеспечения стабильного, структурированного контекста.

- :dfn[План]{canonical="Plan" href="./010_agent_plan.md"}: Контекстное сообщение, содержащее граф потока данных из :term[Вызовов инструментов]{canonical="Tool Call"}, который представляет стратегию агента. Он передаётся между шагами для обеспечения итеративного исполнения и адаптации.

  > Sidenote:
  > - [010: Агент/План](./010_agent_plan.md)

- :dfn[Идея-процесс]{canonical="Process Idea" href="./010*agent_plan.md"}: Самодостаточная [:term[Идея]{canonical="Idea"}](./101_concept_idea.md), которая фиксирует стратегический [:term[План]{canonical="Plan"}](./010_agent_plan.md) и его текущее состояние исполнения. Её `schema` — это библиотека [:term[Инструментов]{canonical="Tool"}](./002_agent_tool.md), её `solution` — это _новый_ `План` для текущего такта, а её `context` содержит `Input`, `State` и предыдущий `Plan`.

  > Sidenote:
  > - [010: Агент/План](./010_agent_plan.md)

- :dfn[Идея-носитель]{canonical="Vessel Idea" href="./202_idea_vessel.md"}: Самодостаточная :term[Идея]{canonical="Idea"}, являющаяся одновременно определением реактивной способности и записью её выбранной реакции. Её `schema` определяет всю вселенную возможных :term[Инструментов]{canonical="Tool"}, а её `solution` фиксирует конкретные :term[Вызовы]{canonical="Call"} (экземпляры этих :term[Инструментов]{canonical="Tool"}), которые были выбраны в ответ на стимул.

  > Sidenote:
  > - [202: Идея/Носитель](./202_idea_vessel.md)

- :dfn[Экземплирование]{canonical="Instancing" href="./011_agent_instancing.md"}: Процесс обработки множества независимых :term[Экземпляров]{canonical="Instance"} (каждый со своим уникальным идентификатором и соответствующим сообщением о :term[состоянии]{canonical="State"}) в рамках одного запроса агента.

  > Sidenote:
  > - [011: Агент/Экземплирование](./011_agent_instancing.md)

- :dfn[Запрос]{canonical="Request" href="./001_agent_request.md"}: Одиночный, самодостаточный вызов LLM, который принимает `context` и `schema` и производит `solution`.

- :dfn[Экземпляр]{canonical="Instance" href="./011_agent_instancing.md"}: Один, уникально идентифицированный объект состояния и связанный с ним контекст, обрабатываемый как один из многих в рамках операции :term[Экземплирования]{canonical="Instancing"}.

  > Sidenote:
  > - [011: Агент/Экземплирование](./011_agent_instancing.md)

- :dfn[Реактор]{canonical="Reactor" href="./303_ideator_reactor.md"}: Особый :term[Трансформер Идей]{canonical="Idea Transformer"}, разработанный как универсальная среда выполнения для пошаговых взаимодействий агентов с сохранением состояния. Он принимает состояние игры/процесса (:term[Идею]{canonical="Idea"}) и производит следующее состояние (:term[Идею]{canonical="Idea"}).

  > Sidenote:
  > - [303: Идеатор/Реактор](./303_ideator_reactor.md)

- :dfn[Улучшатель]{canonical="Refiner" href="./103_concept_ideator.md#the-refiner-an-ideator-for-evolution"}: Специализированный :term[Трансформер Идей]{canonical="Idea Transformer"}, который развивает структуру :term[Идеи]{canonical="Idea"}. Он принимает :term[Идею]{canonical="Idea"} и промпт в качестве входных данных и производит новую :term[Идею]{canonical="Idea"} с обновлённой `schema` и перенесённым `solution`.

- :dfn[Путь поиска]{canonical="Search Path" href="./108_concept_visibility.md#the-search-path-prioritized-retrieval"}: Упорядоченный список имён ветвей, который определяет механизм извлечения. Он указывает резолверу, в каких разделах искать и в каком порядке приоритета, создавая каскадную систему наложения.

- :dfn[Ссылка на переменную]{canonical="Variable Reference" href="./004_agent_call.md"}: Строка со специальным синтаксисом (`†<kind>.<path>`), используемая в параметрах :term[Вызова инструмента]{canonical="Tool Call"} для ссылки на значение из контекста (например, из сообщения о :term[состоянии]{canonical="State"} или :term[входного сообщения]{canonical="Input Message"}). Она позволяет агенту динамически связывать инструменты, используя выход одного инструмента в качестве входа для другого.

- :dfn[Решение]{canonical="Solution" href="./001_agent_request.md"}: Структурированный объект, возвращаемый :term[Запросом]{canonical="Request"}. Он содержит предложенные :term[Вызовы инструментов]{canonical="Tool Call"} и конечный `output`, соответствующий предопределённой JSON Schema.
