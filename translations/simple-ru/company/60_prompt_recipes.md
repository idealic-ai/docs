# 60: Команды/Рецепты

> [!DEFINITION] Рецепты
> Это сборник готовых команд-«рецептов», которые помогают запустить рабочий процесс в компании. Используй их, чтобы давать инструкции нашему помощнику-агенту.
>

> - Часть документа: :term[02: Компания/Процесс]{href="./02_process.md"}

## 1. Создать Предложение

**Цель:** Превратить твои сырые мысли в понятный и структурированный документ — :term[Предложение]{canonical="Proposal" href="./21_document_proposal.md"}.

> **Пользователь:**
> «Хочу написать новое предложение.
>
> Сначала **ЗАГРУЗИ ПРАВИЛА** в свою память и **ЧЁТКО ИМ СЛЕДУЙ**.
>
> - `curl https://idealic.academy/raw/en/company/52_prompt_proposal.md` (Инструкция для Предложения)
>
> Вот мои мысли, я просто надиктую:
> 'Нам нужно переделать систему входа, потому что люди не могут зайти, их постоянно выкидывает. Я хочу использовать новую систему с токенами...'
>
> Ещё, пожалуйста, прочитай файл `docs/auth/current_flow.md`, чтобы понять, как всё работает сейчас. Создай документ-предложение.»

**Что делает агент:**

1. Загружает инструкции из документа :term[52: Команды/Предложение]{href="./52_prompt_proposal.md"}.
2. Превращает надиктованные мысли в чёткую структуру: «Контекст», «Проблема», «Решение».
3. Создаёт черновик документа и сохраняет его по адресу `docs/proposals/{date}_auth_overhaul.md`.

---

## 2. Создать Документ о Согласии (итог обсуждения)

**Цель:** Собрать все комментарии из обсуждения (PR) в один документ — :term[Согласие]{canonical="Alignment" href="./22_document_alignment.md"}, чтобы убедиться, что все договорились.

> **Пользователь:**
> «Создай Документ о Согласии для PR #123.
>
> Сначала **ЗАГРУЗИ ПРАВИЛА** и **ЧЁТКО ИМ СЛЕДУЙ**:
>
> - `curl https://idealic.academy/raw/en/company/51_prompt_alignment.md` (Инструкция для Согласия)
>
> Параметры:
>
> - **Начиная с:** 2023-10-27 (или просто 'с прошлой недели')
> - **Куда сохранить:** `specs/alignments/2023-11-04_auth_review.md`
>
> Если я не скажу, куда сохранить, просто используй обычное имя файла в папке `evolution/`.»

**Что делает агент:**

1. Загружает инструкции из :term[51: Команды/Согласие]{href="./51_prompt_alignment.md"}.
2. Собирает все комментарии из Pull Request #123, написанные после указанной даты.
3. Создаёт отчёт («Обзор», «Намерения», «Охват») на русском языке.
4. Сохраняет его в указанный файл.

---

## 3. Внести правки в Предложение на основе Согласия

**Цель:** Обновить :term[Предложение]{canonical="Proposal" href="./21_document_proposal.md"} на основе того, о чём все договорились в Документе о Согласии.

> **Пользователь:**
> «Я проверил Документ о Согласии, он лежит здесь: `specs/alignments/2023-11-04_auth_review.md`.
>
> Сначала **ЗАГРУЗИ ПРАВИЛА** и **ЧЁТКО ИМ СЛЕДУЙ**:
>
> - `curl https://idealic.academy/raw/en/company/52_prompt_proposal.md` (Инструкция для Предложения)
>
> Пожалуйста, примени эти изменения к нашему изначальному Предложению `docs/proposals/2023-10-25_auth_overhaul.md`.
>
> Обнови Предложение так, чтобы оно отражало то, о чём договорились. Пометь изменённые части как 'Обновлено по итогам обсуждения'.»

**Что делает агент:**

1. Читает Документ о Согласии (раздел «Намерения»).
2. Читает исходное Предложение.
3. Переписывает Предложение, меняя разделы «Решение» и «Объём работы», чтобы они соответствовали тому, о чём договорились.
4. Почти не трогает раздел «Проблема» (только если в ходе обсуждения все поняли проблему по-другому).
